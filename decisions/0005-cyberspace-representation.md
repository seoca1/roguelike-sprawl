# ADR-0005: 사이버스페이스 표현 방식

**상태**: Accepted
**날짜**: 2026-06-17
**결정자**: 사용자
**우선순위**: P1

## 컨텍스트

매트릭스(사이버스페이스)를 게임 안에서 어떻게 표현할지 결정해야 한다. 결정은 다음에 영향을 미친다:
- Pillar 2 (The Matrix) — 매트릭스의 플레이 공간
- 절차적 생성 방식 (노드 그래프 / 미로 / 네트워크)
- Pillar 5 (The Style) — 깁슨 톤
- ADR-0002 (스타일), ADR-0003 (전투)와 연관

## 고려한 옵션

### Option 1: 노드 그래프 (네트워크)

- **설명**: 매트릭스 = 노드(데이터, 시스템, ICE)들의 그래프. 플레이어는 노드 사이를 이동.
- **장점**:
  - 매트릭스의 "데이터 흐름" 표현
  - 절차적 생성과 자연스러움
  - 노드 = 명확한 위치 / 결정점
  - Pillar 2 미학 부합
  - 깁슨의 "lines of light" 묘사와 일치
- **단점**:
  - 미로/탐색의 "물리적 깊이" 표현 어려움
  - 모든 노드가 같아 보일 수 있음
- **Pillar 정합**:
  - P1: 한 노드 = 한 결정
  - P2: "네트워크" 미학
  - P5: 거친 톤
- **메카닉**:
  - 노드 종류: 데이터, ICE, construct, 픽서, 시스템, 파일
  - 노드 사이: 라우터 / 패킷 (비용, 시간, trace 위험)
  - 깊이: 표면 → 미들 → 코어 (깊이 = 위험)

### Option 2: 미로 / 그리드

- **설명**: 매트릭스 = 그리드 / 미로. 플레이어는 타일 사이를 이동.
- **장점**:
  - 클래식 로그라이크
  - FOV, 미로 생성 라이브러리 활용
  - 절차적 생성과 자연스러움
- **단점**:
  - 매트릭스의 "네트워크" 미학 손실
  - 깁슨의 "추상화된 데이터"와 거리
  - "벽"의 개념이 매트릭스에 어색
- **Pillar 정합**:
  - P2: "그리드"는 덜 매트릭스
  - P5: 톤 회피 가능

### Option 3: 하이브리드 (노드 + 그리드)

- **설명**: 노드 = 그리드 셀, 노드 사이 = 거리. "도시"처럼.
- **장점**:
  - 노드 그래프의 추상성 + 그리드의 물리성
  - "도시" 미학 (깁슨: "like city lights, receding")
  - 절차적 생성 + 미니 맵 활용
- **단점**:
  - 구현 복잡
  - 디자인 미묘

### Option 4: 단일 화면 / 절차적 진행

- **설명**: 매트릭스 = 절차적 생성된 단일 화면 (Slay the Spire, FTL).
- **장점**:
  - 단순
  - 한 화면 = 한 노드
  - 절차적 생성과 자연스러움
- **단점**:
  - "항해"의 느낌 약함
  - 매트릭스의 깊이 / 연결성 표현 어려움
  - Pillar 2 약화

### Option 5: 위에서 내려다보는 미니맵 (RTS 스타일)

- **설명**: 매트릭스 = 미니맵 같은 뷰, 노드들이 보이는 형태.
- **장점**:
  - "데이터 흐름" 시각화
  - 한 화면에 큰 그림
- **단점**:
  - 클래식 로그라이크와 다름
  - 1인칭 느낌 약화

## 추천

## 결과 (Consequences)

### 매트릭스 구조

매트릭스 = 노드들의 그래프. 노드 사이 = 라우터 / 패킷 (이동 경로).

> **중요 제약 (2026-06-17)**: ADR-0009에 따라 **meatspace는 절대 시각화되지 않음**. Hub / Job Board / Prep Phase는 모두 cyberspace 안의 텍스트 인터페이스로 표현됨.

**노드 종류**:
- `entry`: 진입점 (의뢰 시작)
- `data`: 데이터 노드 (탈취 가능)
- `system`: 시스템 노드 (해킹 대상)
- `ice`: ICE 노드 (전투)
- `construct`: AI construct (대화)
- `router`: 라우터 (분기점)
- `core`: 코어 노드 (깊이 끝)
- `exit`: 탈출점

**노드 간 이동**:
- `BW` (bandwidth) 소모: 노드 간 이동 = 1 BW
- `AP` 소모: 매크로 액션 (예: 빠른 이동)
- `Trace 위험`: 깊은 노드로 갈수록 증가

**깊이 / 존**:
- 표면 (Surface): 평이한 시스템, 낮은 알람
- 미들 (Mid): 기업 시스템, 중간 알람
- 코어 (Core): 치명적 시스템, 높은 알람

### ASCII 표현

```
                [System: S1]
                  |
            [Router: R1]
            /         \
       [Data: D1]   [ICE: I1]
         |             |
       [Exit: X1]   [Core: C1]
```

- 노드: `[X: 라벨]`
- 연결: `|` (수직), `/` `\` (대각)
- 미발견: `?` 또는 빈 공간
- 플레이어 위치: `*` 또는 `@`

### Pillar 정합
- P1 (The Run): 한 노드 = 한 결정
- P2 (The Matrix): "네트워크" 미학
- P5 (The Style): 거친 톤

### 구현 영향
- 노드 그래프 = `dict` 또는 NetworkX 사용
- 절차적 생성: 의뢰마다 노드 그래프 새로 생성
- 깊이 = 데이터 의존

### 향후 결정
- 노드 그래프 생성 알고리즘 (BSP, WFC, 단순 random)
- 노드 간 이동 비용 (BW/AP)
- 깊이 시스템 수치
- 미니맵 표현 (그래프 전체 보기)
