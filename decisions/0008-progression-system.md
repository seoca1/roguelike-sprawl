# ADR-0008: 진행 / 레벨업 시스템

**상태**: Accepted (Revised 2026-06-17)
**날짜**: 2026-06-17 (Revised 2026-06-17)
**결정자**: 사용자
**우선순위**: P1

## 컨텍스트

게임의 "레벨업" 메카닉을 결정해야 한다. 결정은 다음에 영향을 미친다:
- Pillar 1 (The Run) — 한 런의 텐션 곡선
- Pillar 4 (The Build) — 메타 진행
- 플레이어 진행감
- 깁슨 톤 — "레벨"은 사이버펑크답지 않을 수 있음

## 고려한 옵션

### Option 1: 런 내 스탯 성장 없음 (스킬 = 장비)

- **설명**: 플레이어의 "능력"은 런 동안 변하지 않음. 변하는 것은 **장비 / 프로그램 / 웨웨어 / construct**.
- **장점**:
  - Pillar 1 (The Run)의 일관성 — 한 런 동안 강해지지 않음
  - Pillar 4 (The Build) — 메타 진행이 unlock
  - 깁슨 톤 — 자키는 도구로 강해짐, 스탯으로 아님
  - 절차적 생성과 자연스러움
- **단점**:
  - 일부 플레이어에게 "성장감" 부족
  - 런 후반이 너무 어려울 수 있음
- **Pillar 정합**:
  - P1: ⭐⭐⭐⭐⭐
  - P4: ⭐⭐⭐⭐⭐
  - P5: ⭐⭐⭐⭐⭐

### Option 2: 런 내 XP / 레벨

- **설명**: 일반 RPG 스타일. 경험치 획득 → 레벨업 → 스탯 증가.
- **장점**:
  - 성장감
  - 보상 명확
- **단점**:
  - 깁슨 톤과 거리 ("level"은 판타지)
  - Pillar 1 약화 — 런 후반이 쉬워짐
  - 비-기둥 (grind) 위험
- **Pillar 정합**:
  - P1: ⭐⭐ 약화
  - P4: 강제 메타 진행

### Option 3: 런 내 도구 성장 (장비 강화 / 프로그램 업그레이드)

- **설명**: 런 내에서는 장비 / 프로그램의 강화로 성장. 스탯은 변하지 않음.
- **예시**:
  - 의뢰 완료 → 프로그램 강화
  - 노드 발견 → 웨웨어 모듈 추가
  - construct와 결합 → 능력 확장
- **장점**:
  - 성장감 + 깁슨 톤
  - Pillar 1 유지 (강해지지만, 도구로)
  - 절차적 생성과 잘 맞음
- **단점**:
  - 디자인 복잡
  - 밸런스 어려움
- **Pillar 정합**:
  - P1: ⭐⭐⭐⭐
  - P4: ⭐⭐⭐⭐
  - P5: ⭐⭐⭐⭐⭐

### Option 4: Construct와의 결합 (런 내)

- **설명**: 런 내에서 construct와 결합할수록 능력 확장. construct는 메타 진행.
- **장점**:
  - construct 활용
  - 깁슨 톤 (Dixie 류)
  - Pillar 4 자연스러움
- **단점**:
  - construct 디자인 필요
  - 의존성

### Option 5: 자키 등급 (1-up / 2-up / 3-up)

- **설명**: 자키는 등급이 있음 (1-up = 초보, 5-up = 전설). 등급은 메타 진행. 등급이 높을수록 시작 장비 다양.
- **장점**:
  - 깁슨 원작의 자키 등급 시스템
  - 메타 진행의 명확한 표현
  - Pillar 4 자연스러움
- **단점**:
  - 등급 디자인 필요
  - "더 좋은 시작"이 Pillar 1 약화 가능 (의도된 디자인이지만)

## 추천

**Option 1: 런 내 스탯 성장 없음** + **Option 3: 런 내 도구 성장** 조합
**Option 5: 자키 등급** (메타 진행)

근거:
- 깁슨 톤 — 자키는 도구로 강해짐
- Pillar 1, 4 자연스러움
- 메타 진행은 자키 등급 (Option 5)
- 런 내 도구 강화 (Option 3)로 성장감 보완

## 디자인 권장안

### 런 내 (Pillar 1)
- **스탯 변화 없음**
- **장비 / 프로그램 / 웨웨어 / construct**가 변화
- **의뢰 완료 / 노드 발견** → 도구 unlock 또는 강화 (런 내)
- **결합 (binding)** — 런 내 결정으로 도구 결합 / 업그레이드

### 메타 진행 (Pillar 4)
- **자키 등급** (1-up ~ 5-up) — 메타 unlock
- **새 도구 unlock** (영구)
- **새 construct unlock** (영구)
- **새 의뢰 라인 unlock** (영구)

### 등급 (예시)
| 등급 | 설명 | 시작 도구 |
| --- | --- | --- |
| 1-up | 신참 | Ono-Sendai 4, 기본 프로그램 2개 |
| 2-up | 일반 | Ono-Sendai 5, 기본 프로그램 4개 |
| 3-up | 숙련 | Ono-Sendai 6, 확장 프로그램 4개 |
| 4-up | 베테랑 | Ono-Sendai 7, 고급 프로그램 4개 |
| 5-up | 전설 | Ono-Sendai 7, 풀 액세스 |

(원작의 SAMURAI 등급 컨셉 차용)

## 결과 (Consequences)

### 런 내 (Pillar 1)
- **스탯 변화 없음** (HP, BW, PW, HD 등)
- **장비 / 프로그램 / 웨웨어 / construct** 변화
- **의뢰 완료 / 노드 발견** → 도구 unlock 또는 강화 (런 내)
- **결합 (binding)** — 런 내 결정으로 도구 결합 / 업그레이드

### 레벨업 = 아이템/장비 티어 (사용자 확인, 2026-06-17)

사용자 확인: "레벨업 요소를 장비나 아이템에 구현해도 좋아"

- **레벨업 = 아이템/장비 티어 (Item Tier)**
- **티어 시스템**: T1 (Basic) ~ T5 (Legendary)
- **티어가 높을수록 강함**
- 캐릭터의 "성장"은 *도구의 성장*
- XP / 레벨 시스템 X
- 스탯 누적 X (Pillar 3)
- 런 시작 시 동일 (등급 외, Pillar 1)

#### 티어 예시

| 티어 | 데크 | 프로그램 | 웨웨어 | 예시 |
| --- | --- | --- | --- | --- |
| T1 | Ono-Sendai 4 | Basic Wisp | Standard | 신참 |
| T2 | Ono-Sendai 5 | Wisp + Hammer | Standard | 일반 |
| T3 | Ono-Sendai 6 | Goliath + Wisp | Biochip | 숙련 |
| T4 | Ono-Sendai 7 | Goliath + Wardrone | Advanced | 베테랑 |
| T5 | Ono-Sendai 7 | Kraken + All | Premium + Construct | 전설 |

#### 획득 = 레벨업

- 의뢰 완료 → 보상 = 더 높은 티어의 장비
- 노드 발견 → 새 프로그램
- Construct 해제 → 메타 진행
- **매번 "더 좋은 도구"를 얻는 것이 "레벨업"**
- **장비 교체 = 레벨 변경**

#### 전투 영향 (ADR-0003 연동)

- Tier ↑ → AP ↑, HP ↑, BW ↑, PW ↑
- 더 높은 티어 program unlock (Goliath, Kraken 등)
- Combat에서 "더 강한 스킬" = "레벨업 결과"

### 메타 진행 (Pillar 4)
- **자키 등급** (1-up ~ 5-up) — 메타 unlock의 척도
- **새 도구 unlock** (영구) — 등급 상승으로 사용 가능
- **새 construct unlock** (영구)
- **새 의뢰 라인 unlock** (영구)
- **스탯 영구 강화 X** — Pillar 3 유지

### 자키 등급 시스템 (예시)

| 등급 | 설명 | 시작 도구 | 의뢰 풀 |
| --- | --- | --- | --- |
| 1-up | 신참 | Ono-Sendai 4, 기본 프로그램 2개 | 평이한 의뢰만 |
| 2-up | 일반 | Ono-Sendai 5, 기본 프로그램 4개 | + 일반 의뢰 |
| 3-up | 숙련 | Ono-Sendai 6, 확장 프로그램 4개 | + 기업 의뢰 |
| 4-up | 베테랑 | Ono-Sendai 7, 고급 프로그램 4개 | + 위험 의뢰 |
| 5-up | 전설 | Ono-Sendai 7, 풀 액세스 | 모든 의뢰 |

(원작의 SAMURAI 등급 컨셉 차용)

### 등급 상승 조건
- N개 의뢰 성공
- 특정 의뢰 라인 클리어
- 또는 단순 누적 (런 횟수)

### Pillar 정합
- P1 (The Run): 동일 시작점 (등급 외)
- P3 (The Flatline): "더 강해지지 않음" 유지
- P4 (The Build): unlock으로 보상
- P5 (The Style): "더 많은 도구"는 OK

### 구현 영향
- 자키 프로필 = `{grade, unlocks: [...]}`
- 매 런 시작: 등급 + unlock 사용
- 사망 시: unlock 유지, 자키 영구 종료 (새 자키 시작)
- 등급 시스템 = JSON 데이터 + 검증 로직

### 향후 결정
- 등급 상승 조건 (정확한 수치)
- 등급별 unlock 매핑
- "자키 5-up 달성 시" 엔드 콘텐츠 (T-A 라인?)
- **티어 시스템 밸런스** — T1~T5 구체적 수치 (HP, AP, BW, PW, 데미지)
- **티어별 program 해금** — 어느 티어에서 어떤 program 사용 가능
- **티어 보상 분포** — 의뢰 보상이 어느 티어의 아이템을 주는지
